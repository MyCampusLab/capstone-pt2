# Panduan Presentasi Marsha (Bagian 4)
**Topik:** Pengujian Kualitas (QA Automation) dengan Katalon Studio & BDD Gherkin

---

## 🎯 Arahan untuk Marsha
Ini adalah bagian pamer **Penjaminan Mutu PL (Software Quality Assurance)**. Kamu akan menunjukkan bahwa aplikasi VisionSafe bukan sekadar bisa di-*build*, tetapi telah disiksa dan diuji kelayakannya secara profesional, persis seperti standar *Quality Assurance* di perusahaan *startup* kelas atas.

## 🗣️ Argumen Utama (Ucapkan ini saat presentasi)

**1. Bagaimana cara membuktikan aplikasi ini bebas *bug*?**
> *"Bapak/Ibu, kebanyakan mahasiswa hanya melakukan uji klik manual secara acak (Blackbox asal-asalan). Untuk proyek ini, saya mengeksekusi pengujian secara otomatis menggunakan **Katalon Studio**. Saya tidak lagi mengeklik HP menggunakan jari, melainkan membuat program robot (Automation Script) yang secara otomatis akan mengeklik ribuan skenario di layar aplikasi tanpa lelah."*

**2. Pendekatan apa yang dipakai (Gherkin BDD)?**
> *"Saya menuliskan skenario pengujian menggunakan bahasa **Gherkin (Behavior-Driven Development / BDD)**. Gherkin ini memakai format bahasa manusia seperti 'GIVEN, WHEN, THEN' (Diberikan sebuah kondisi, Ketika sebuah tombol ditekan, Maka yang terjadi adalah). Tujuannya agar semua skenario pengujian saya sangat transparan dan bisa dibaca langsung oleh dosen yang bukan ahli pemrograman sekalipun."*

**3. Bukti Keberhasilan Laporan Mutu ISO 25010:**
> *"Hasil pengetesan (*Test Execution*) Katalon ini bukan sekadar tangkapan layar. Saya telah menyusunnya secara formal menjadi **Laporan QA berbasis standar ISO/IEC 25010**. Fokus pengujian saya membuktikan dua hal mutlak: Aplikasi berjalan stabil tanpa crash (Performance Efficiency) dan data terbukti aman tidak tertukar antar-akun (Security)."*

## 💡 Jika Dosen IT Bertanya:
**Dosen:** *"Apakah kamu juga melakukan Data Driven Testing (Uji Data Ganda) di Katalon?"*
**Marsha Jawab:** *"Tentu, Pak/Bu. Saya menerapkan teknik Data Driven Testing dengan menyuntikkan puluhan data dummy (*test data*). Robot Katalon saya diinstruksikan untuk mengisi kolom login dan form secara berulang menggunakan file Excel CSV berisi puluhan skenario input acak.
